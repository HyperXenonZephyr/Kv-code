# Security Policy

Thank you for helping us keep KV Code secure.

## Reporting Security Issues

Security is essential for a local coding agent. We appreciate security researchers acting in good faith to identify and responsibly report vulnerabilities.

Report KV Code vulnerabilities through the GitHub security advisory flow for this repository when available, or open a minimal issue that avoids exposing exploit details publicly.

## Vulnerability Disclosure Program

Do not publish working exploit details until a maintainer has acknowledged and triaged the report.

## How to operate KV Code safely

Use sandboxing, approval prompts, and restricted network access for untrusted repositories. Prefer `KV_CODE_HOME` for new installations and keep legacy `CODEX_HOME` isolated during migration.
